# Issue tracker: GitHub Issues

Issues for this repo live in **GitHub Issues** on `PatrickvBeek/portfolioTracker`.

## Workflow

- **Create issue**: `gh issue create`
- **List issues**: `gh issue list`
- **View issue**: `gh issue view <number>`
- **Close issue**: `gh issue close <number>`
- **Reopen issue**: `gh issue reopen <number>`
- **Add labels**: `gh issue edit <number> --add-label <label>`
- **Remove labels**: `gh issue edit <number> --remove-label <label>`
- **Comment**: `gh issue comment <number> --body "<text>"`

## Conventions

- Use conventional-commit-style titles where practical.
- Link PRs to issues using `Fixes #<number>` or `Closes #<number>` in commit messages.
